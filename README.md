# Accounting Fundamentals Online Examination System

ระบบสอบออนไลน์วิชาการบัญชีเบื้องต้น สำหรับนักเรียนชั้น ม.4/3 และ ม.4/4 โรงเรียนเซนต์เทเรซา เชื่อมฐานคะแนนกลางกับ Google Sheet โดยไม่เก็บรายชื่อนักเรียนหรือรหัสผ่านไว้ในรีโพสาธารณะ

## คุณสมบัติ

- ข้อสอบ 5 ชุด ชุดละ 40 ข้อ ข้อละ 1 คะแนน
- สุ่มชุด ลำดับข้อ และลำดับตัวเลือกเฉพาะผู้เข้าสอบ
- จับเวลา 40 นาที บันทึกคำตอบระหว่างทำ และส่งอัตโนมัติเมื่อหมดเวลา
- กรอกเลขประจำตัวนักเรียนแล้วเติมชื่อ–นามสกุล เลขที่ และห้องจากทะเบียนอัตโนมัติ
- กลับเข้าทำต่อได้หลังรีเฟรช โดยใช้ attempt token ของเครื่องเดิม
- กันสอบซ้ำจากเลขประจำตัวด้วย Script Lock และตรวจซ้ำที่ฐานข้อมูลกลาง
- ตรวจคำตอบและคำนวณคะแนนบนเซิร์ฟเวอร์ ไม่ส่งเฉลยจริงไปยังเบราว์เซอร์
- หน้าครูค้นหาและกรองตามห้อง/ชุดข้อสอบ
- ส่งออก CSV และไฟล์ Excel-compatible `.xls`
- รองรับคอมพิวเตอร์ แท็บเล็ต และโทรศัพท์ โดยบนโทรศัพท์แถบเวลาและสถานะจะติดด้านบน และรายการ 40 ข้อพับ–ขยายได้

## ไฟล์สำหรับ Google Apps Script

| ไฟล์ในโฟลเดอร์ | ชื่อไฟล์ใน Apps Script | หน้าที่ |
| --- | --- | --- |
| `Code.gs` | `Code.gs` | API, สุ่มข้อสอบ, ตรวจคะแนน, Google Sheet, สิทธิ์ครู |
| `Questions.gs` | `Questions.gs` | คลังข้อสอบ 5 ชุดและตัวเลือก |
| `index.html` | `index.html` | โครงหน้าเว็บและ template includes |
| `style.html` | `style.html` | CSS ที่ใช้ใน Web App |
| `app.html` | `app.html` | JavaScript ฝั่งผู้ใช้ |
| `appsscript.json` | `appsscript.json` | manifest และ timezone |

`app.js` และ `style.css` ใช้ทดสอบหน้าจอในเครื่อง โดยเนื้อหาตรงกับ `app.html` และ `style.html`

## วิธีติดตั้ง

1. สร้างหรือเปิด Google Sheet สำหรับเก็บข้อมูล แล้วคัดลอก Spreadsheet ID จาก URL
2. สร้าง/แทนที่ไฟล์ใน Apps Script ให้ตรงกับตารางด้านบน
3. ใน `Code.gs` เปลี่ยน `CONFIG.SPREADSHEET_ID` และ `CONFIG.INITIAL_TEACHER_PIN` เป็นค่าจริง โดยรหัสครูควรมีอย่างน้อย 8 ตัวอักษร
4. เลือกฟังก์ชัน `setupSystem` แล้วกด **Run** หนึ่งครั้ง จากนั้นอนุญาตสิทธิ์เข้าถึงชีต
5. ระบบจะสร้างแท็บ `ผลสอบ`, แท็บซ่อน `คำตอบชั่วคราว` และแท็บ `ทะเบียนนักเรียน` พร้อมหัวตารางอัตโนมัติ
6. วางข้อมูลนักเรียนลงในแท็บ `ทะเบียนนักเรียน` ตามลำดับคอลัมน์ที่อธิบายด้านล่าง
7. เลือก **Deploy → New deployment → Web app**
8. ตั้ง **Execute as: Me** และ **Who has access: Anyone** แล้วกด Deploy
9. ส่ง URL ที่ลงท้ายด้วย `/exec` ให้นักเรียนใช้สอบ

> อย่าเผยแพร่ซอร์สโค้ด Apps Script หรือให้สิทธิ์แก้ไข Google Sheet แก่นักเรียน นักเรียนควรได้รับเฉพาะ URL ของ Web App เท่านั้น

## เปลี่ยนรหัสครูภายหลัง

แก้ `CONFIG.INITIAL_TEACHER_PIN` ใน `Code.gs` แล้วรันฟังก์ชัน `resetTeacherPinFromConfig` หนึ่งครั้ง จากนั้นสร้าง deployment เวอร์ชันใหม่

## ทดสอบหน้าจอในเครื่อง

เปิด `index.html?demo=1` ผ่าน local web server เช่น VS Code Live Server โหมดสาธิตไม่เขียน Google Sheet และใช้ข้อมูลจำลองเท่านั้น

- รหัสครูในโหมดสาธิต: `2569demo`
- หากเปิด `index.html` โดยไม่มี `?demo=1` ระบบจะแสดงคำแนะนำให้เปิดผ่าน Apps Script Web App

## โครงสร้างฐานข้อมูล

- `ผลสอบ`: วันเวลา, เลขประจำตัว, ชื่อ–นามสกุล, เลขที่, ห้อง, ชุดข้อสอบ, คะแนน, เวลาที่ใช้, สถานะส่งอัตโนมัติ
- `คำตอบชั่วคราว`: attempt token, เวลาเริ่ม/กำหนดส่ง, คำตอบ JSON, สถานะ และเวลาบันทึกล่าสุด โดยแท็บนี้ถูกซ่อนจากมุมมองปกติ
- `ทะเบียนนักเรียน`: เลขประจำตัว, ชื่อ–นามสกุล, เลขที่, ห้อง และสถานะ

## ทะเบียนนักเรียน

หน้าเว็บจะขอข้อมูลเฉพาะเลขประจำตัวที่กรอกตรงกัน และเซิร์ฟเวอร์จะตรวจทะเบียนซ้ำก่อนเริ่มสอบ ข้อมูลในแท็บต้องเรียงเป็น:

| เลขประจำตัว | ชื่อ–นามสกุล | เลขที่ | ห้อง | สถานะ |
| --- | --- | --- | --- | --- |
| `99999` | `นักเรียน ตัวอย่าง` | `18` | `ม.4/3` | `กำลังศึกษาอยู่` |

สถานะที่อนุญาตคือ `กำลังศึกษาอยู่`, `นักเรียนเข้าใหม่`, `active` หรือเว้นว่าง หากนำเข้าจาก Excel ควรเลือกเฉพาะคอลัมน์ที่จำเป็นและไม่อัปโหลดข้อมูลส่วนบุคคลลง GitHub
